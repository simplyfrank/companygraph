# User Journeys: KPI Definition, OKR Settings, and Roll-Down

## Overview

This document outlines the user journeys for KPI definition, OKR setting, and domain/product-level roll-down across three organizational levels:
- **Executive Leadership** (C-level, VP level)
- **Senior Management** (Domain Directors, Product Owners)
- **Program Steering** (Program Managers, Steering Committee)

---

## 1. KPI Definition User Journey

### 1.1 Executive Leadership

**Goal**: Define and approve organization-wide KPIs that align with strategic objectives.

**User Journey Steps**:

1. **Strategic Planning Phase**
   - Review organizational strategy and strategic objectives
   - Identify critical success factors for the organization
   - Define KPI categories (financial, operational, customer, quality, time)
   - Set KPI hierarchy (leading vs lagging indicators)

2. **KPI Definition**
   - Access CompanyGraph → Executive Dashboard → KPI Management
   - Create new KPI:
     - Name and description
     - Category selection
     - Unit of measurement
     - Target value and direction (higher/lower is better)
     - Warning and critical thresholds
     - Measurement frequency (daily, weekly, monthly, quarterly)
     - Owner role assignment
   - Define KPI calculation methodology
   - Set data source and collection method

3. **Alignment Review**
   - Review KPI alignment with strategic objectives
   - Validate KPI measurability and data availability
   - Assess KPI relevance and actionability
   - Approve or reject KPI definition

4. **Domain Assignment**
   - Assign KPIs to relevant domains
   - Set domain-specific targets if needed
   - Define attribution weights for domain contributions

5. **Monitoring and Review**
   - Monitor KPI performance via Executive Dashboard
   - Review KPI trends and variances
   - Adjust KPI targets or definitions as needed
   - Annual KPI review cycle

**Key Screens/Components**:
- Executive KPI Dashboard
- KPI Definition Form
- KPI Alignment Matrix
- KPI Performance Trends

**Success Criteria**:
- KPIs are measurable and data-driven
- KPIs align with strategic objectives
- KPIs have clear ownership and accountability
- KPIs are actionable and drive decision-making

---

### 1.2 Senior Management (Domain Directors)

**Goal**: Define domain-specific KPIs and align organizational KPIs to domain operations.

**User Journey Steps**:

1. **Domain KPI Planning**
   - Review organizational KPIs assigned to domain
   - Identify domain-specific success factors
   - Define domain-level KPIs that complement organizational KPIs
   - Assess current domain performance gaps

2. **KPI Definition**
   - Access CompanyGraph → Domain Detail → KPIs Tab
   - Create domain-specific KPIs:
     - Name and description
     - Category and unit
     - Target value and direction
     - Thresholds and frequency
     - Owner assignment (domain role)
   - Align domain KPIs to organizational KPIs

3. **Journey/Activity Alignment**
   - Align KPIs to relevant user journeys
   - Align KPIs to specific activities within journeys
   - Set attribution weights for journey/activity contributions
   - Define measurement points in the journey

4. **Target Setting**
   - Set domain-specific targets for organizational KPIs
   - Set targets for domain-specific KPIs
   - Define baseline values
   - Set improvement targets over time

5. **Monitoring and Adjustment**
   - Monitor domain KPI performance via Domain Dashboard
   - Review KPI performance against targets
   - Identify underperforming areas
   - Adjust targets or interventions as needed
   - Monthly/quarterly review cycle

**Key Screens/Components**:
- Domain KPI Dashboard
- KPI Alignment Form (to journeys/activities)
- KPI Target Setting
- KPI Performance Reports

**Success Criteria**:
- Domain KPIs align with organizational KPIs
- KPIs are relevant to domain operations
- KPIs have clear measurement points in journeys
- KPIs drive domain improvement initiatives

---

### 1.3 Program Steering

**Goal**: Define program-level KPIs and track program progress against organizational and domain KPIs.

**User Journey Steps**:

1. **Program KPI Planning**
   - Review organizational and domain KPIs relevant to program
   - Define program-specific KPIs (milestone completion, budget, quality)
   - Identify program success criteria
   - Map program KPIs to organizational/domain KPIs

2. **KPI Definition**
   - Access CompanyGraph → Program Detail → KPIs Tab
   - Create program KPIs:
     - Milestone completion rate
     - Budget variance
     - Quality metrics
     - Timeline adherence
   - Link program KPIs to domain KPIs

3. **Baseline and Target Setting**
   - Establish program baselines
   - Set program targets aligned with domain targets
   - Define interim milestones and targets
   - Set warning thresholds

4. **Monitoring and Reporting**
   - Monitor program KPI performance
   - Generate program KPI reports
   - Report progress to steering committee
   - Identify risks and mitigation actions

5. **Adjustment and Course Correction**
   - Adjust program targets based on changes
   - Reallocate resources based on KPI performance
   - Update program plans as needed
   - Communicate changes to stakeholders

**Key Screens/Components**:
- Program KPI Dashboard
- Program KPI Reports
- Steering Committee Reports
- Risk and Mitigation Tracking

**Success Criteria**:
- Program KPIs track program health and progress
- Program KPIs align with domain and organizational KPIs
- KPIs enable timely course correction
- KPIs support stakeholder communication

## 2. OKR Setting User Journey

### 2.1 Executive Leadership

**Goal**: Set organization-wide OKR cycles and approve strategic objectives.

**User Journey Steps**:

1. **OKR Cycle Planning**
   - Define OKR cycle cadence (quarterly, semi-annual, annual)
   - Set OKR cycle dates (start, end, review dates)
   - Define OKR themes for the cycle
   - Align OKR cycle with fiscal planning cycle

2. **Strategic Objective Setting**
   - Access CompanyGraph → Executive Dashboard → OKRs Tab
   - Create OKR Directive for the cycle:
     - Cycle name and description
     - Cycle dates
     - Review cadence
     - Status (draft, active, review, closed)
   - Define strategic objectives:
     - Objective name and description
     - Owner assignment (executive sponsor)
     - Theme/category
     - Expected outcomes

3. **Key Result Definition**
   - Define key results for each objective:
     - KR name and description
     - Baseline value
     - Target value
     - Current value
     - Unit of measurement
     - Direction (higher/lower is better)
     - Progress tracking
     - Status (not started, in progress, achieved, at risk, missed)
   - Link key results to organizational KPIs
   - Set baseline values from KPI historical data

4. **Review and Approval**
   - Review OKR alignment with strategy
   - Validate KR measurability
   - Assess KR ambition and achievability
   - Approve OKR cycle for activation

5. **Monitoring and Review**
   - Monitor OKR progress via Executive Dashboard
   - Review OKR performance at cycle checkpoints
   - Identify at-risk key results
   - Adjust OKRs if strategic priorities change
   - Close cycle and evaluate outcomes

**Key Screens/Components**:
- Executive OKR Dashboard
- OKR Directive Creation Form
- Objective and Key Result Forms
- OKR Performance Board
- OKR Review Reports

**Success Criteria**:
- OKRs align with strategic objectives
- Key results are measurable and time-bound
- OKRs drive organizational focus
- OKR progress is visible and actionable

---

### 2.2 Senior Management (Domain Directors)

**Goal**: Set domain-level OKRs aligned with organizational OKRs and KPIs.

**User Journey Steps**:

1. **Domain OKR Planning**
   - Review organizational OKRs for the cycle
   - Identify domain contributions to organizational objectives
   - Define domain-specific objectives
   - Assess domain capacity and resources

2. **OKR Directive Creation**
   - Access CompanyGraph → Domain Detail → OKRs Tab
   - Create domain OKR Directive:
     - Link to organizational OKR cycle
     - Domain-specific cycle dates
     - Domain review cadence
   - Define domain objectives:
     - Objective name and description
     - Owner (domain director or domain role)
     - Theme aligned with organizational theme

3. **Key Result Definition**
   - Define domain key results:
     - KR name and description
     - Baseline, target, current values
     - Unit and direction
     - Progress and status
   - Link domain KRs to:
     - Organizational KRs (contribution)
     - Domain KPIs (evidence)
   - Set domain-specific baselines and targets

4. **Alignment and Validation**
   - Validate domain OKR alignment with organizational OKRs
   - Ensure KRs are achievable with domain resources
   - Get buy-in from domain teams
   - Approve domain OKRs

5. **Monitoring and Management**
   - Monitor domain OKR progress via Domain Dashboard
   - Review OKR performance at domain checkpoints
   - Identify at-risk KRs and interventions
   - Update progress and current values
   - Communicate progress to executive leadership

**Key Screens/Components**:
- Domain OKR Dashboard
- Domain OKR CRUD Interface
- OKR-KPI Alignment View
- OKR Performance Board
- Progress Update Forms

**Success Criteria**:
- Domain OKRs align with organizational OKRs
- Domain KRs are achievable and measurable
- OKRs drive domain focus and improvement
- Progress is visible and communicated

## 3. Domain-Level Roll-Down User Journey

### 3.1 Executive Leadership

**Goal**: Roll down organizational KPIs and OKRs to domains and ensure alignment.

**User Journey Steps**:

1. **Roll-Down Planning**
   - Review organizational KPIs and OKRs
   - Identify domains that contribute to each KPI/OKR
   - Define domain contribution weights
   - Set domain-specific targets

2. **KPI Roll-Down**
   - Access CompanyGraph → Executive Dashboard → Roll-Down View
   - For each organizational KPI:
     - Select contributing domains
     - Set domain contribution weights
     - Set domain-specific targets
     - Define domain attribution rules
   - Validate roll-down totals match organizational targets

3. **OKR Roll-Down**
   - For each organizational OKR:
     - Identify contributing domains
     - Define domain-level objectives
     - Set domain-level key result targets
     - Link domain KRs to organizational KRs
   - Validate alignment and achievability

4. **Communication and Buy-In**
   - Communicate roll-down to domain directors
   - Explain targets and expectations
   - Get domain director commitment
   - Address concerns and adjustments

5. **Monitoring and Adjustment**
   - Monitor domain contributions to organizational KPIs/OKRs
   - Review roll-down effectiveness
   - Adjust weights or targets if needed
   - Support domains in achieving targets

**Key Screens/Components**:
- Executive Roll-Down Dashboard
- KPI Roll-Down Matrix
- OKR Roll-Down Matrix
- Domain Contribution Reports
- Roll-Down Validation Tools

**Success Criteria**:
- Roll-down is transparent and understood
- Domain targets are achievable
- Roll-down totals match organizational targets
- Domains are accountable for contributions

---

### 3.2 Senior Management (Domain Directors)

**Goal**: Receive organizational roll-down, set domain targets, and roll down to teams/journeys.

**User Journey Steps**:

1. **Receive Roll-Down**
   - Review organizational KPI/OKR roll-down
   - Understand domain contribution requirements
   - Assess domain capacity and constraints
   - Identify gaps and risks

2. **Domain Target Setting**
   - Access CompanyGraph → Domain Detail → Roll-Down Tab
   - Review domain-specific targets
   - Validate achievability
   - Negotiate adjustments if needed
   - Commit to domain targets

3. **Journey/Activity Roll-Down**
   - For each domain KPI:
     - Identify contributing journeys
     - Set journey contribution weights
     - Set journey-specific targets
     - Align to activities within journeys
   - For each domain OKR:
     - Identify contributing teams/journeys
     - Set team-level objectives
     - Define team-level key results
     - Link to journey/activities

4. **Team Communication**
   - Communicate roll-down to teams
   - Explain targets and expectations
   - Get team commitment
   - Provide support and resources

5. **Monitoring and Support**
   - Monitor journey/activity contributions
   - Track team OKR progress
   - Identify underperforming areas
   - Provide coaching and support
   - Report progress to executive leadership

**Key Screens/Components**:
- Domain Roll-Down Dashboard
- Journey Contribution Matrix
- Team OKR Setting
- Progress Tracking
- Domain Performance Reports

**Success Criteria**:
- Domain targets are achieved
- Journey/activity contributions are tracked
- Teams are aligned and committed
- Progress is visible and actionable

## 5. Cross-Cutting Considerations

### 5.1 Data Flow and Integration

**KPI Data Flow**:
- Organizational KPIs → Domain KPIs → Journey/Activity KPIs
- KPI measurements flow up the hierarchy
- Attribution weights determine contribution calculations

**OKR Data Flow**:
- Organizational OKRs → Domain OKRs → Team OKRs
- Key result progress flows up
- KR-KPI links provide evidence

**Roll-Down Data Flow**:
- Executive sets organizational targets
- Domains receive and commit to targets
- Domains roll down to journeys/activities
- Progress flows back up for monitoring

### 5.2 Approval Workflows

**KPI Definition Approval**:
- Domain KPIs: Domain Director approval
- Organizational KPIs: Executive approval
- Program KPIs: Program Manager + Steering Committee approval

**OKR Setting Approval**:
- Domain OKRs: Domain Director approval
- Organizational OKRs: Executive approval
- Program OKRs: Program Manager + Steering Committee approval

**Roll-Down Approval**:
- Domain roll-down: Domain Director commitment
- Journey roll-down: Team lead commitment
- Product roll-down: Product Owner commitment

### 5.3 Review Cycles

**KPI Review Cycles**:
- Daily: Operational KPIs
- Weekly: Tactical KPIs
- Monthly: Strategic KPIs
- Quarterly: KPI target adjustments

**OKR Review Cycles**:
- Weekly: OKR progress check-ins
- Monthly: OKR status reviews
- Quarterly: OKR cycle close and evaluation

**Roll-Down Review Cycles**:
- Monthly: Domain progress review
- Quarterly: Roll-down effectiveness review
- Annually: Strategic alignment review

### 5.4 Notifications and Alerts

**KPI Alerts**:
- Warning threshold breaches
- Critical threshold breaches
- Target achievement
- Trend anomalies

**OKR Alerts**:
- At-risk key results
- Missed key results
- Progress milestones
- Cycle deadlines

**Roll-Down Alerts**:
- Target achievement
- Contribution shortfalls
- Risk indicators
- Deadline reminders
